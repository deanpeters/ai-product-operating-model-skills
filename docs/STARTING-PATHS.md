# Choose Your AIPOM Starting Path

You do not need to read or run all 41 skills. Start with the decision you need to make.

## 1. Assess an Organization

Choose this path when leaders need an evidence-based view of how the organization currently chooses, builds, evaluates, governs, and learns from AI product work.

Start with [Assess an Organization](../commands/aipom-assess-organization.md).

Expected result: a seven-category maturity profile, critical gaps, prioritized interventions, owners, and an operating-model roadmap.

## 2. Evaluate an AI Initiative

Choose this path for a specific AI idea, pilot, launch, scale, sensitive-data, autonomy, or funding decision.

Start with [Evaluate an AI Initiative](../commands/aipom-evaluate-initiative.md).

Expected result: a framed opportunity, owned bet, evidence and readiness profile, critical constraints, and a proceed, constrain, remediate, pause, or stop decision.

## 3. Redesign a Product-Team Workflow

Choose this path when a recurring product-team decision or productive motion is slow, inconsistent, context-poor, or being changed through AI assistance.

Start with [Redesign a Product-Team Workflow](../commands/aipom-redesign-workflow.md).

Expected result: an observed current-state motion, explicit human-AI responsibilities, a tested redesign, measures, and a reusable playbook.

## If You Are Unsure

- Choose organization assessment when the problem spans several teams or categories.
- Choose initiative evaluation when a specific investment or launch decision is pending.
- Choose workflow redesign when the unit of change is recurring work performed by a product team.

Start smaller when evidence, ownership, or capacity is weak. A focused path that changes one decision is more useful than running the entire library as a checklist.
