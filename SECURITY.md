# Security and Sensitive Reports

Do not disclose credentials, tokens, active exploits, private repository information, personal data, security architecture, or other security-sensitive details in a public issue.

Use [GitHub private vulnerability reporting](https://github.com/deanpeters/ai-product-operating-model-skills/security/advisories/new) for software-security concerns.

Use the public **Potentially unsafe or unauthorized recommendation** issue form for sanitized method-safety concerns, such as a skill hiding a critical gap, assigning unsupported authority, or recommending consequential action without necessary evidence or controls.

This repository is primarily Markdown methods and lightweight build scripts. Security reports should identify the affected file or release artifact, the consequence, and a safe reproduction path without exposing real secrets or confidential organizational material.
