# Worked Example

**Synthetic example.** A software portfolio has 14 pilots, nine using similar vendor copilots. Only three have named outcomes and none has stop criteria. One support pilot has representative evidence; two customer-facing agents lack behavior contracts.

The advisor recommends: selectively scale the support pilot after economics review; validate two workflow bets; contain vendor concentration with a dependency audit; pause the two agents pending behavior and autonomy controls; stop five duplicate pilots; and require bet charters for the remainder. The portfolio leader owns decisions at a 60-day review.
