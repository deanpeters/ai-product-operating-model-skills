# Synthetic Worked Example: Entitlement Context

Entitlement records are active only while the account system confirms status; generated case briefs are derived and expire after the case changes. A corrected entitlement propagates to open briefs and invalidates cached outputs. Conflicting contract and account records move context to disputed, display both sources, and block automated routing. Restricted customer notes never enter reusable context. Tests cover stale caches, correction propagation, missing sources, permission loss, and retirement after case closure.
